{# Apply / Manager actions #}

Apply udev sticky names

/etc/udev/rules.d/99-pipineapple-adapters.rules

Writes a udev rules file from the role assignments above and triggers a udev reload. The new names take effect on next interface add — either unplug+replug each Alfa, or reboot the Pi.

Apply NM unmanaging config

/etc/NetworkManager/conf.d/99-pipineapple-unmanaged.conf

Tells NetworkManager to leave wlan-mon-* and wlan-ap alone (matches the sticky names above) while continuing to manage wlan0 for upstream Wi-Fi.

Stop NetworkManager + wpa_supplicant

one-time sledgehammer — equivalent of airmon-ng check kill

Kills both services until next boot. Will disconnect wlan0 from your home Wi-Fi (ensure you're on Ethernet for the management plane). Prefer the NM unmanaging config above for everyday use — this is the "everything off" option for legacy attacks that need it.
