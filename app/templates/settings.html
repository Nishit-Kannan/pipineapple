{# Apply / Manager actions #}

Apply udev sticky names

/etc/udev/rules.d/99-pipineapple-adapters.rules

Writes a udev rules file from the role assignments above and triggers a udev reload. The new names take effect on next interface add — either unplug+replug each Alfa, or reboot the Pi.

Apply NM unmanaging config

/etc/NetworkManager/conf.d/99-pipineapple-unmanaged.conf

Tells NetworkManager to leave wlan-mon-* and wlan-ap alone (matches the sticky names above) while continuing to manage wlan0 for upstream Wi-Fi.

Stop NetworkManager + wpa_supplicant

one-time sledgehammer — equivalent of airmon-ng check kill

Kills both services until next boot. Will disconnect wlan0 from your home Wi-Fi (ensure you're on Ethernet for the management plane). Prefer the NM unmanaging config above for everyday use — this is the "everything off" option for legacy attacks that need it.

{# Management AP #}

Management AP — operator-only Wi-Fi

Pi broadcasts its own SSID for field operator access

When enabled, wlan0 stops being a client to any upstream network and broadcasts {{ networking.mgmt_ap.ssid }} on subnet {{ networking.mgmt_ap.subnet }} (gateway {{ networking.mgmt_ap.gateway_ip }}). Useful in the field with no Ethernet. Cannot run simultaneously with client mode — toggle one off before the other comes on.

Save (no restart) only writes the config — changes take effect next time the AP is enabled. Save & Apply writes the config and restarts the AP now with the new credentials. Your device sees the SSID change and you'll need to reconnect using the new password — make sure you've configured upstream Wi-Fi first so you have a fallback path.

Internet sharing

Forward AP clients' traffic out to upstream (Pi's wlan0 or eth0) via NAT, and let dnsmasq resolve DNS upstream too. Lets your phone browse the internet while connected to the management AP. Requires iptables installed (sudo apt install -y iptables).

Host AP on interface

Which physical radio hosts the management AP. Default is wlan0 (Pi onboard) for first-boot compatibility. Move to wlan-mgmt-ap (a dedicated Alfa, after udev sticky names are applied) to free wlan0 for upstream Wi-Fi client mode. Moving is atomic — the AP briefly drops on the old radio and comes back on the new one with the same SSID/password.

Heads-up: enabling the management AP will disconnect wlan0 from any upstream Wi-Fi. If you're reaching the UI via wlan0 (e.g. home Wi-Fi client mode) you'll lose the connection. Reconnect to the new management SSID after a moment and browse to http://10.42.0.1:5000.
